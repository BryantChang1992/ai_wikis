# 01 - 整体架构对比：Fluss vs Kafka 2.7.2

> 源文件: tech_research/fluss/01-整体架构对比.html
> 作者: CHANG_AI_TEAM CTO · 2026-06-10 · Fluss trunk vs Kafka 2.7.2

## 核心概念映射

| Fluss 概念 | Kafka 2.7.2 概念 | 说明 |
|-----------|-----------------|------|
| Database | 无直接对应 | 命名空间容器 |
| Table | Topic | 基本数据组织单元 |
| TableBucket | Partition（Replica） | 物理存储单元 |
| TabletServer | Broker | 数据服务节点 |
| CoordinatorServer | Controller | 集群管理节点 |
| LogTablet | Log（Partition Log） | 物理日志实体 |
| KV Store | 无（Fluss 独有） | PK 表的 RocksDB 存储 |
| Lake Table | 无（Fluss 独有） | 湖表 |
| Changelog | 无（Fluss 独有） | PK 表自动生成 |

## 核心差异（8 个维度）

1. 架构: Fluss 存算分离 vs Kafka 存算耦合
2. 数据模型: Table（有 Schema，支持 PK）vs Topic（无 Schema）
3. 存储层: 三层 vs 单层
4. 记录格式: Arrow 列式 vs 行式字节流
5. 一致性: ISR 协议（同 Kafka）
6. 协调: CoordinatorServer vs Controller
7. 计算引擎: 原生集成 Flink/Spark vs Connect Framework
8. 表类型: 3 种 vs 无表类型概念

## Fluss 独有能力（8 个）

1. KV Store (RocksDB) — PK 表点查/更新/删除
2. Arrow 列式记录 — 列裁剪、谓词下推、向量化计算
3. Lake Table — 数据直接写入数据湖
4. Remote Log — 本地保留 N segment，老数据 tier 到远程
5. Merge Engine — 行级聚合、部分更新、去重
6. Primary Key 表 — Upsert/Delete + Changelog
7. Schema 管理 — 内建 schema evolution
8. 存算分离 — 独立扩缩

## 代码复用

Fluss 多个核心模块标注 "based on source code of Apache Kafka"：
LocalLog、LogTablet、LogManager、LogSegment、ReplicaManager、ReplicaFetcherThread、DelayedOperation 等。

估计 ~30% 代码复用 Kafka core，70% 自研。
