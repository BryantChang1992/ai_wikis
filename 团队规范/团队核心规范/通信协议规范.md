# 通信协议规范

## 核心原则

1. **常驻 Agent 之间一律用 `sessions_send`**
2. **临时 worker 用 `sessions_spawn` `isolated`**
3. **不跨层通信**（VP 不直接指挥别家的专家，上级不绕过中间层直接指挥底层）

## 常驻 Agent 清单

| 层级 | Agent | 数量 |
|------|-------|------|
| CEO | CEO (Mike) | 1 |
| VP | CTO, CPO, CFO, COO | 4 |
| 专家（CTO 线） | PMO, RD 专家, 性能专家, QA 专家, SRE 专家 | 5 |
| 专家（其他 VP 线） | 用户研究, 产品设计, 预算, 审计, 税务, 数据分析, 流程优化 | 7 |

## 通信矩阵

| 场景 | 方式 | 说明 |
|------|------|------|
| CEO → VP | `sessions_send` | 常驻间 |
| VP ↔ 同级 VP | `sessions_send` | 常驻间 |
| CTO → 专家（RD/Perf/QA/SRE/PMO） | `sessions_send` | 常驻间 |
| VP → 本级常驻专家 | `sessions_send` | 常驻间 |
| 专家 ↔ 同级专家 | `sessions_send` | 常驻间 |
| 专家 → worker | `sessions_spawn` `isolated` | **唯一 spawn 场景** |
| CTO 紧急跨层 → worker | `sessions_spawn` `isolated` | 允许但需同步通知专家 |

## 模型分配

- 常驻 Agent（VP/专家层）：`deepseek/deepseek-v4-pro`
- 临时 Worker：`deepseek/deepseek-v4-flash`

## sessions_spawn 知识传递

子 Agent 不知道父 Agent 上下文。派发任务时需在 task 中嵌入知识引用（告知查阅什么规范文件）。
