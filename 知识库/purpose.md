# 知识库的目的

## 为什么存在

CHANG_AI_TEAM 是一个 AI Agent 团队，每天产生大量技术调研、架构决策、工程实践和踩坑经验。这些知识如果散落在各个会话、各个 Worker 的记忆中，就无法形成积累。知识库是团队的**持久化大脑**——LLM 编译一次，持续使用，不断更新。

## 核心目标

1. **积累**：每次调研、每次决策、每次踩坑都沉淀为结构化知识，不丢失
2. **关联**：知识点之间用 [[wikilink]] 建立关系网，形成可导航的知识图谱
3. **可用**：Agent 和人类都能快速找到需要的信息，不靠记忆靠索引
4. **演进**：知识不是一次性的——新发现会修正旧认知，旧结论会被标注为过时

## 当前研究重点

- **分布式系统**：事务模型、一致性协议、存储引擎
- **数据平台**：OLAP 引擎（Doris）、流存储（Fluss/Kafka/AutoMQ）、Lakehouse
- **AI 基础设施**：Agent 框架、Memory、RAG、模型评测
- **工程实践**：性能优化、可观测性、部署运维

## 使用方式

- **人类**：通过 Obsidian 浏览、搜索、编辑，利用 Graph View 发现关联
- **Agent**：写入前先读 `purpose.md` 和 `schema.md`，理解上下文和规则
- **新成员**：从这里开始，了解团队知识体系全貌
