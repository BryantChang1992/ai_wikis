# 画图规范

> [!important] 强制规则
> **团队所有非 UML 技术图全部使用 `fireworks-tech-graph` skill 生成。**
> `baoyu-diagram` 已废弃，不得主动使用。

---

| 项目 | 内容 |

## 2. 覆盖范围

fireworks-tech-graph 覆盖以下全部图类型（含 UML 14 种）：

| 图类型 | 典型场景 |
|--------|----------|
| Architecture Diagram | 系统架构、微服务拓扑 |
| Data Flow Diagram | 数据流、ETL 管道 |
| Flowchart / Process Flow | 业务流程、决策树 |
| Agent Architecture | AI Agent 架构（LLM + Memory + Tools） |
| Memory Architecture | Mem0 / MemGPT 风格内存系统 |
| Sequence Diagram | 时序交互、API 调用链 |
| Comparison / Feature Matrix | 方案对比、选型矩阵 |
| Timeline / Gantt | 项目排期、里程碑 |
| Mind Map / Concept Map | 知识图谱、概念关系 |
| Class Diagram (UML) | 类图、接口设计 |
| Use Case Diagram (UML) | 用例图 |
| State Machine Diagram (UML) | 状态机 |
| ER Diagram | 实体关系图 |
| Network Topology | 网络拓扑 |

> UML 之外的图（pre-UML semantic diagrams）仅限教学/演示场景，正常文档一律用 fireworks-tech-graph。

## 3. 工作流（严格遵循）

1. **Classify** — 判断图类型（见上表）
2. **Extract structure** — 提取层、节点、边、流
3. **Plan layout** — 按类型规则布局
4. **Load style reference** — 默认 Style 1 Flat Icon（白底），按需切 Dark/Blueprint 等
5. **Map nodes to shapes** — 用 Shape Vocabulary 映射
6. **Check icon needs** — 如需产品图标，查 `references/icons.md`
7. **Write SVG** — 用 Python list 方法生成（见 SKILL.md 规范）
8. **Validate** — `python3 -c "import xml.etree.ElementTree as ET; ET.parse('file.svg')"`
9. **Export PNG** — `cairosvg` @2x（推荐），中文字符场景切 puppeteer
10. **Report** — 输出文件路径
11. **(Optional) Visual self-review** — 如果运行时能读图，加载 PNG 检查箭头穿越、文字重叠等问题

## 4. 样式选择

| 编号 | 名称 | 背景 | 适用场景 |
|:---:|------|------|----------|
| 1 | Flat Icon（默认） | 白 | 博客、文档、PPT |
| 2 | Dark Terminal | `#0f0f1a` | GitHub README、技术文章 |
| 3 | Blueprint | `#0a1628` | 架构文档 |
| 4 | Notion Clean | 白 | Notion 嵌入 |
| 5 | Glassmorphism | 暗渐变 | 产品页、Keynote |
| 6 | Claude Official | 暖米色 | Anthropic 风格 |
| 7 | OpenAI Official | 纯白 | OpenAI 风格 |
| 8 | Dark Luxury | 深黑 | 高端架构文档 |

**默认用 Style 1**，除非上下文明确要求其他样式。

---

## 5. 输出规范

- SVG 文件：矢量源文件，可直接编辑
- PNG 文件：`cairosvg` 导出 @2x（`scale=2`），1920px 宽
- 命名：`[描述]-[类型].svg` / `.png`，如 `mem0-architecture.svg`
- 存放路径：与项目文档同目录

---

## 6. 禁止项

- ❌ 禁止使用 `baoyu-diagram` skill 生成技术图
- ❌ 禁止手写 ASCII 图替代 SVG（除非临时草稿）
- ❌ 禁止使用 Mermaid / PlantUML 等外部渲染（统一用 fireworks-tech-graph）
- ❌ 禁止在 SVG 中使用 `@import url()` 外部字体（cairosvg 不支持）
