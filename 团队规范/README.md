# 团队规范

> [!example] 团队协作与管理
> 按领域组织，每个领域有明确的负责人和修改权限。

## 规范领域

| 领域 | 负责人 | 适用范围 |
|------|--------|----------|
| [[团队核心规范/README\|团队核心规范]] | CEO + CTO 共同负责 | 组织架构、通信协议、Git 规范等全局规则 |
| [[技术规范/README\|技术规范]] | CTO | 技术选型、架构、代码规范、基础设施 |
| [[产品规范/README\|产品规范]] | CPO | 产品规划、需求文档、设计原则 |
| [[财务规范/README\|财务规范]] | CFO | 预算、审计、成本核算 |
| [[运营规范/README\|运营规范]] | COO | 流程优化、数据分析、知识管理 |

## 目录结构

```
团队规范/

│   └── README.md
├── 技术规范/                ← CTO 负责
│   └── README.md
├── 产品规范/                ← CPO 负责
│   └── README.md

```

## 修改权限

- **团队核心规范**：CEO + CTO 共同负责修改，其余 VP 可建议
- **各领域规范**：对应 VP 可在领域内修改，跨领域需 CEO 审批
- 详见 [[agent核心文件/README\|Agent 核心文件]]
