# 知识管理规范

## 知识沉淀路径

| 产出类型 | 写入路径 |
|---------|---------|
| 技术决策（选型、架构变更） | `work/ai_wikis/知识库/` |
| 新流程/方法/规范 | `work/ai_wikis/团队规范/` 对应领域 |
| 调研/分析结果 | `work/ai_wikis/技术文章/` |

## 沉淀流程

1. 任务完成后检查是否满足沉淀条件
2. 写入对应路径
3. `git add → commit → push`
4. PMO 定期审查知识新鲜度

## PMO 职责

PMO 是知识沉淀的最终守门人：
- 审查所有 Agent 知识沉淀完整性
- 维护 ai_wikis 索引与分类
- 每周检查知识新鲜度，标记过期文档
