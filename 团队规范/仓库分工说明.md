# 仓库分工说明

## 概述

CHANG_AI_TEAM 维护两个 Git 仓库，分别服务于内部知识管理和对外发布。

## 仓库清单

| 项目 | 仓库名 | 用途 | 访问方式 |
|------|--------|------|----------|
| Obsidian Vault | `ai_wikis` | 团队内部知识库 | Obsidian 客户端 + GitHub |
| 博客/站点 | `ai_memory_chang_ai_team` | 对外技术博客 & 设计文档 | GitHub Pages |

## 1. ai_wikis（Obsidian Vault）

- **远程**: `git@github.com:BryantChang1992/ai_wikis.git`
- **本地路径**: `work/ai_wikis/`
- **用途**: 团队内部知识库，用 Obsidian 打开管理
- **特点**: 私有 repo，存放内部规范、会议记录、OKR、技术文档草稿

### 目录结构

```
ai_wikis/
├── 团队规范/       ← 团队流程、角色定义、基础设施文档
├── 技术文章/       ← 技术分析、调研
├── 知识库/         ← 技术知识点沉淀
└── 项目文档/       ← 各项目设计文档
    └── agent基础设施可观测性平台/
```

## 2. ai_memory_chang_ai_team（GitHub Pages）

- **远程**: `git@github.com:BryantChang1992/ai_memory_chang_ai_team.git`
- **本地路径**: `work/ai_memory_chang_ai_team/`
- **用途**: 对外公开的技术博客 & 设计文档站点
- **特点**: 通过 GitHub Pages 自动部署为静态网站

### 目录结构

```
ai_memory_chang_ai_team/
├── index.html                    ← 首页
├── memory/                       ← 博客文章
│   ├── domains/
│   ├── projects/
│   ├── shared/
│   └── vp/
├── tech_designs/                 ← 设计文档
│   ├── agent-infra/
│   ├── diagram/
│   ├── images/
│   └── infographic/
├── tech_research/                ← 技术调研周报
│   ├── ai_harness/
│   ├── data_for_ai/
│   └── kafka_research/
└── scripts/                      ← 辅助脚本
```

## 工作流程

1. **内部文档** → 先在 `ai_wikis` 中编写、讨论、完善
2. **对外发布** → 定稿后同步到 `ai_memory_chang_ai_team` 发布

## Git 操作规范

> [!warning] 强制遵守
> 每次操作任一仓库文件前，**必须先 `git pull`**；操作完成后，**必须自动 `git add -A && git commit && git push`**。
